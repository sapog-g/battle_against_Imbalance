# Битва с дисбалансом
**Дата:** Ноябрь 2025  
**Организатор:** Газпром Нефть

## Описание Проекта
В рамках спецтрека от Газпром Нефти мы решали задачу многоклассовой классификации обезличенных корпоративных писем. Главный вызов проекта заключался не в сложности модели, а в **критическом дисбалансе классов** и необходимости глубокой предобработки сырых данных.

**Философия решения:** Качество данных важнее сложности модели. Наша цель — подготовить идеальный датасет, на котором даже простая модель покажет результат выше Baseline.

## Задачи
- **Многоклассовость:** Большое количество категорий, по которым нужно распределить письма.
- **Дисбаланс:** Экстремально неравномерное распределение примеров по классам.
- **Target Metrics:** 
  - Максимизация **Macro F1-score**.
  - Минимизация **дисперсии F1-score** (равномерность качества по всем классам).
- **Baseline (BERT):** Macro F1 = 0.46, Variance = 0.099.

## Наша стратегия обработки данных
В рамках исследования мы прошли полный цикл подготовки данных:
1. **Глубокий EDA:** Анализ распределения длин писем, частоты слов и специфики терминологии по каждому классу.
2. **Текстовая предобработка:** Нормализация, очистка от шума и приведение текстов к формату, оптимальному для векторизации.
3. **Борьба с дисбалансом:** Применение техник Oversampling/Undersampling и синтетической генерации признаков для выравнивания весов малых классов.
4. **Iterative Modeling:** Валидация качества данных на простых классификаторах для быстрой проверки гипотез перед финальной выгрузкой.

## Командная работа
Проект реализован командой **George**. 
В данном репозитории представлена финальная презентация нашего решения, описывающая логику предобработки и бизнес-эффект от внедрения.

**Мой вклад:**
- Анализ дисбаланса классов: Выявление критического разброса в данных (от 2 до 199 писем на класс, разница в 100 раз). Оценка покрытия датасета (Топ-10 классов занимают 73% данных).
- Анализ геометрических свойств текстов: Исследование распределения длин писем. Обнаружение длинного хвоста (максимум 69к символов) и обоснование рисков обрезки текста для модели BERT (лимит токенов).
- Детекция текстовых артефактов: Поиск повторяющихся технических блоков, которые создают шум для классификатора.
- Эксперименты с Oversampling, то есть сравнение двух подходов: агрессивный - дублирование объектов до 30-40 экземпляров в классе, мягкий - минимальная поддержка хвоста до 8 экземпляров.
